同步/异步和串行/并发

1.同步串行

死锁原因：队列引起的循环等待，不是线程引起的循环等待。
在主队列，先提交了一个viewDidLoad任务，然后又提交了一个block任务。这里任务都要分派到主线程中去执行。现在分派viewDidLoad到主线程中执行，在执行过程中，需要调用block，当block同步调用完成之后，viewDidLoad才能往下走。viewDidLoad调用结束依赖后续提交的block任务。而block执行，依赖队列的性质(主队列性质)，先进先出的性质。等到viewDidLoad完成才能处理block提交。所以，block提交依赖viewDidLoad完成。因此，产生了相互等待，死锁。

3.异步串行

示例

1.

同步方式提交任务，无论是在串行队列，还是并行队列，都是在当前线程运行(即：不会开启新的线程)。

如果：这列global_queue都换成自定义的串行队列，会产生死锁。

2.

异步方式分派到全局并发队列中，本身block会在GCD所有维护的线程池中的某个线程上执行。而GCD底层分派的这些线程默认没有开启对应的RunLoop,而performSelector:withObject:afterDealy:,要创建这个相应一个提交任务到RunLoop上面的一个逻辑。在GCD底层创建线程没有RunLoop的情况下，performSelector:withObject:afterDealy:会失效。即performSelector:withObject:afterDealy:方法有效，必须是方法调用所属的当前线程是有RunLoop。

dispatch_group_async()


@interface GroupObject()
{
    dispatch_queue_t concurrent_queue;
    NSMutableArray <NSURL *> *arrayURLs;
}

@end

@implementation GroupObject

- (id)init
{
    self = [super init];
    if (self) {
        // 创建并发队列
        concurrent_queue = dispatch_queue_create("concurrent_queue", DISPATCH_QUEUE_CONCURRENT);
        arrayURLs = [NSMutableArray array];
    }

    return self;
}

- (void)handle
{
    // 创建一个group
    dispatch_group_t group = dispatch_group_create();

    // for循环遍历各个元素执行操作
    for (NSURL *url in arrayURLs) {

        // 异步组分派到并发队列当中
        dispatch_group_async(group, concurrent_queue, ^{

            //根据url去下载图片

            NSLog(@"url is %@", url);
        });
    }

    dispatch_group_notify(group, dispatch_get_main_queue(), ^{
        // 当添加到组中的所有任务执行完成之后会调用该Block
        NSLog(@"所有图片已全部下载完成");
    });
}
